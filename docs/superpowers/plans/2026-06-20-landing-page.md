# Landing Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the `App.tsx` placeholder with the designed Peakstats landing page, and establish the project-wide tooling (Tailwind v4, shadcn/ui, Recharts, Leaflet) that all future phases build on.

**Architecture:** `LandingPage.tsx` is a single self-contained component styled with Tailwind utility classes. Dark/light mode is class-based (`.dark` on `document.documentElement`), matching shadcn's convention. The dashboard preview card uses a Recharts `AreaChart` with static mock data. `App.tsx` renders `<LandingPage />` directly; routing is deferred.

**Tech Stack:** Tailwind CSS v4, `@tailwindcss/vite`, shadcn/ui, Recharts, Leaflet + `@types/leaflet`, Vitest + React Testing Library, Google Fonts (Space Grotesk, Archivo, JetBrains Mono)

## Global Constraints

- Strava orange: `#fc4c02` — exposed as `bg-strava` / `text-strava` via `@theme`
- Dark mode class: `.dark` on `document.documentElement` (not `data-theme`)
- CTA `href="#"` only — no backend wiring
- Default theme: dark (add `class="dark"` to `<html>` in `index.html`)
- Max-width: 1240px; breakpoints: 900px (single-column), 560px (compact padding)
- All copy is metric (km, m, km/h) on the landing page
